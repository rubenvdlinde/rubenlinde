# Contributing to Ruben van der Linde's Website

Thank you for your interest in contributing! While this is primarily a personal website, I welcome suggestions, bug reports, and improvements.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in the [Issues](https://github.com/rubenvdlinde/rubenlinde/issues) section
2. If not, create a new issue with:
   - A clear title and description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Content

Have an idea for a blog post topic? Feel free to:

- Open an issue with the tag 'content-suggestion'
- Describe the topic and why it would be interesting

### Code Contributions

If you would like to contribute code:

1. Fork the repository
2. Create a new branch (`git checkout -b feature/improvement`)
3. Make your changes
4. Test locally (`npm start` and `npm run build`)
5. Commit your changes (`git commit -m 'Add some improvement'`)
6. Push to the branch (`git push origin feature/improvement`)
7. Open a Pull Request

### Pull Request Guidelines

- Keep changes focused on a single issue
- Update documentation if needed
- Follow the existing code style
- Test your changes locally before submitting

## Development Setup

## Code of Conduct

Be respectful and constructive in all interactions. This is a welcoming space for everyone interested in learning and sharing knowledge.

## Questions?
